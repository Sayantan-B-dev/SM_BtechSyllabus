### Topic Recap: Data Abstraction, Data Independence, DDL vs DML

**Data Abstraction Levels**:
- **Physical Level**: How data is stored (blocks, pages, indexes).
- **Logical Level**: What data is stored and relationships.
- **View Level**: What users see (subset of data).

**Data Independence**:
- **Logical**: Change logical schema without affecting external views.
- **Physical**: Change physical storage without affecting logical schema.

**DDL (Data Definition Language)**: CREATE, ALTER, DROP, TRUNCATE
**DML (Data Manipulation Language)**: SELECT, INSERT, UPDATE, DELETE

### Problem 1: Data Independence Scenarios

For each change, state whether it is logical or physical data independence:

| Change | Type |
|--------|------|
| Adding a new column to a table | Logical independence (views unaffected) |
| Creating a new index on a column | Physical independence (queries unchanged) |
| Splitting a table into two tables | Logical independence (views abstract the change) |
| Changing file organization from heap to B+ tree | Physical independence |
| Adding a new table to the database | Logical independence |
| Changing data type of a column | Logical independence (views may need modification) |

### Problem 3: View Level Example

A university database has the logical schema:

**Student**(RollNo, Name, DOB, Phone, Email, Address, EnrollmentYear, FeesBalance)
**Course**(CourseCode, Title, Credits, Instructor)
**Enrollment**(RollNo, CourseCode, Semester, Grade)

Create two external views:

(a) **StudentGrades**: Shows RollNo, Name, CourseCode, Grade (for students to check grades)
(b) **FacultyView**: Shows CourseCode, Title, StudentCount, AvgGrade (for faculty)

**Solution**:

(a) StudentGrades view:
```sql
CREATE VIEW StudentGrades AS
SELECT s.RollNo, s.Name, e.CourseCode, e.Grade
FROM Student s, Enrollment e
WHERE s.RollNo = e.RollNo;
```

(b) FacultyView view:
```sql
CREATE VIEW FacultyView AS
SELECT e.CourseCode, c.Title,
       COUNT(e.RollNo) AS StudentCount,
       AVG(e.Grade) AS AvgGrade
FROM Enrollment e, Course c
WHERE e.CourseCode = c.CourseCode
GROUP BY e.CourseCode, c.Title;
```
