### Topic Recap: Data Models, Integrity Constraints

**Data Models**:
- **ER Model**: Entity-Relationship (entities, attributes, relationships).
- **Network Model**: Records linked via pointers. Complex.
- **Hierarchical Model**: Tree-like structure (parent-child).
- **Relational Model**: Data in tables (rows and columns). Most widely used.
- **Object-Oriented Model**: Objects with attributes and methods.

**Integrity Constraints**:
- **Domain Constraint**: Attribute values must be from a domain.
- **Key Constraint**: Primary key must be unique and not null.
- **Entity Integrity**: Primary key cannot be null.
- **Referential Integrity**: Foreign key must match a primary key value or be null.
- **Check Constraint**: Custom condition on column values.

### Problem 3: Integrity Constraints

Specify the appropriate integrity constraints for a Student table:

```sql
CREATE TABLE Student (
    RollNo INT PRIMARY KEY,             -- Entity integrity: unique, not null
    Name VARCHAR(50) NOT NULL,           -- Domain: NOT NULL
    Age INT CHECK (Age >= 16 AND Age <= 100),  -- Check constraint
    Gender CHAR(1) CHECK (Gender IN ('M', 'F', 'O')),  -- Domain constraint
    DeptCode VARCHAR(5) REFERENCES Department(DeptCode) -- Referential integrity
);
```

Violations:
- INSERT INTO Student VALUES (NULL, 'Alice', 20, 'F', 'CS') -> Primary key cannot be null
- INSERT INTO Student VALUES (101, NULL, 20, 'F', 'CS') -> Name cannot be null
- INSERT INTO Student VALUES (101, 'Alice', 150, 'F', 'CS') -> Age check fails
- INSERT INTO Student VALUES (101, 'Alice', 20, 'X', 'CS') -> Gender domain violation
- INSERT INTO Student VALUES (101, 'Alice', 20, 'F', 'XYZ') -> Referential integrity (if no Dept XYZ)

### Problem 4: Relational Model Concepts

Convert this ER concept to relational model:

**Entity**: Employee(EmpID, Name, Salary, DeptID)
**Entity**: Department(DeptID, DeptName, Location)
**Relationship**: Employee works in Department (N:1)

**Relational Schema**:

```
Employee(EmpID: INT PK, Name: VARCHAR(50), Salary: DECIMAL(10,2), DeptID: INT FK -> Department)
Department(DeptID: INT PK, DeptName: VARCHAR(50), Location: VARCHAR(50))
```

Foreign key DeptID in Employee references DeptID in Department.
Each employee belongs to one department. One department has many employees.

### Problem 5: Hierarchical vs Network vs Relational

Compare the three models:

| Feature | Hierarchical | Network | Relational |
|---------|-------------|---------|------------|
| Structure | Tree | Graph (sets) | Tables |
| Relationship | 1:N (parent-child) | M:N (sets) | Via foreign keys |
| Navigation | Top-down | Through pointers | Declarative (SQL) |
| Data independence | Low | Low | High |
| Complexity | Simple queries only | Complex navigation | Easy queries |
| Example | IMS | IDMS, CODASYL | Oracle, MySQL |
