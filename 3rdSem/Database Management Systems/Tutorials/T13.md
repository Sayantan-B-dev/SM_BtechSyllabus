### Topic Recap: Database Security Threats, CIA Triad, Authentication

**CIA Triad**:
- **Confidentiality**: Only authorized users can access data.
- **Integrity**: Data is accurate and not tampered with.
- **Availability**: Data is accessible when needed.

**Security Threats**:
- Accidental losses (human error, natural disasters)
- Theft and fraud
- SQL injection
- Malware and ransomware
- Insider threats
- Denial of Service (DoS)

**Authentication**: Verifying user identity (passwords, biometrics, MFA).

### Problem 2: SQL Injection Example

Vulnerable query:
```sql
query = "SELECT * FROM Users WHERE username='" + username + "' AND password='" + password + "'";
```

If username = `admin' --`, the query becomes:
```sql
SELECT * FROM Users WHERE username='admin' --' AND password='anything'
```

The `--` comments out the rest. The attacker logs in as admin without knowing the password.

**Prevention**: Use parameterized queries (prepared statements).

```python
cursor.execute("SELECT * FROM Users WHERE username=? AND password=?", (username, password))
```

Now the input is treated as data, not SQL code.

### Problem 4: Access Control Models

Compare access control models:

**DAC (Discretionary Access Control)**:
- Owner of data controls access.
- Grant/revoke privileges.
- Example: SQL GRANT and REVOKE.
- Risk: Users can propagate access.

**MAC (Mandatory Access Control)**:
- System-wide policy enforced by the DBMS.
- Labels (Top Secret, Secret, Confidential, Unclassified).
- No user discretion.
- Used in military/government.

**RBAC (Role-Based Access Control)**:
- Permissions assigned to roles, not individuals.
- Users assigned to roles.
- Example: DBA, Developer, Analyst roles.

```sql
CREATE ROLE Analyst;
GRANT SELECT ON Employee TO Analyst;
GRANT SELECT ON Department TO Analyst;
GRANT Analyst TO Alice;
```

### Problem 5: GRANT and REVOKE

Example privilege management in SQL:

```sql
-- Grant SELECT on Student to user1
GRANT SELECT ON Student TO user1;

-- Grant all privileges on Course to user2 with grant option
GRANT ALL PRIVILEGES ON Course TO user2 WITH GRANT OPTION;

-- Grant INSERT on specific columns
GRANT INSERT (Name, Dept) ON Student TO user3;

-- Revoke SELECT from user1
REVOKE SELECT ON Student FROM user1;

-- Revoke all privileges from user2 (cascading)
REVOKE ALL PRIVILEGES ON Course FROM user2 CASCADE;
```
