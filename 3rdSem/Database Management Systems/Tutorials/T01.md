## Notes

### Topic Recap: Introduction to DBMS, File-System vs DBMS

**Key Concepts**:
- **Database**: Organized collection of structured data.
- **DBMS**: Software to manage databases (Oracle, MySQL, PostgreSQL).
- **File System**: Data stored in flat files. Prone to redundancy, inconsistency, and security issues.
- **Advantages of DBMS**: Reduced redundancy, data consistency, security, concurrent access, backup/recovery.

### Problem 1: File System vs DBMS

For each scenario, state whether a file system or DBMS is more appropriate and why.

| Scenario | Choice | Reason |
|----------|--------|--------|
| Personal grocery list | File system | Simple, single-user, no concurrency needed |
| University student records | DBMS | Multiple users, complex queries, security |
| Bank transaction system | DBMS | ACID compliance, concurrent access, recovery |
| System configuration file | File system | Simple key-value, single user |
| E-commerce product catalog | DBMS | Complex queries, many users, relationships |
| Daily temperature log | File system | Simple append-only data |

### Problem 2: DBMS Applications

Identify the DBMS role in each application:

1. **Airline Reservation System**: DBMS manages bookings, seat availability, passenger data, and handles concurrent reservations without double-booking.

2. **Hospital Management**: DBMS stores patient records, doctor schedules, prescriptions, billing. Ensures data privacy and integrity.

3. **Library Management**: DBMS tracks books, members, borrowing history. Enforces constraints (max books per member, due dates).

4. **Social Media Platform**: DBMS stores user profiles, posts, comments, likes. Handles millions of concurrent users.

### Problem 3: Identify Redundancy

The following data is stored in two files in a file system. Identify redundancy and potential inconsistency:

**Student file**: (RollNo, Name, Phone, Course, CourseInstructor, InstructorPhone)
**Course file**: (CourseCode, CourseName, InstructorName, InstructorPhone, Department)

**Solution**:

Redundancies:
- InstructorPhone stored in both files
- InstructorName appears in both (as CourseInstructor and InstructorName)

Inconsistency risk:
- If an instructor's phone changes, it must be updated in both files. If only one is updated, data becomes inconsistent.

DBMS solution: Store instructor details in a separate table, reference by ID.

### Problem 4: DBMS Components

| Component | Function |
|-----------|----------|
| Query Processor | Interprets and executes SQL queries |
| Storage Manager | Manages data storage on disk |
| Transaction Manager | Ensures ACID properties |
| Lock Manager | Handles concurrency control |
| Recovery Manager | Restores database after failure |
| Buffer Manager | Caches data in memory for fast access |
