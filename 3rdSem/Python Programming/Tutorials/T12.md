### Topic Recap: File Handling Wrap-Up, Exception Handling Basics

**File Handling Recap**:
- `os.path.exists(path)` — check if file exists
- `os.remove(path)` — delete a file
- Using `try/except` with file operations to handle errors.

**Exceptions**:
- Runtime errors that disrupt normal program flow.
- Common exceptions: `ZeroDivisionError`, `ValueError`, `FileNotFoundError`, `TypeError`, `IndexError`, `KeyError`
- `try/except/else/finally` blocks:
  - `try`: code that may raise an exception.
  - `except`: handle the exception.
  - `else`: runs if no exception occurred.
  - `finally`: always runs (cleanup).

### Problem 1: Safe File Reading with Error Handling

Write a program that safely reads a file and handles errors.

```python
filename = input("Enter filename to read: ")
try:
    with open(filename, "r") as f:
        print(f.read())
except FileNotFoundError:
    print("Error: File '" + filename + "' not found.")
except PermissionError:
    print("Error: Permission denied to read '" + filename + "'.")
except Exception as e:
    print("An unexpected error occurred: " + str(e))
```

**Output:**
```
Enter filename to read: nonexistent.txt
Error: File 'nonexistent.txt' not found.
```

### Problem 2: Handling Multiple Exceptions

Write a program that handles division by zero and invalid input.

```python
def safe_divide():
    try:
        a = float(input("Enter numerator: "))
        b = float(input("Enter denominator: "))
        result = a / b
    except ValueError:
        print("Error: Please enter valid numbers.")
    except ZeroDivisionError:
        print("Error: Division by zero is not allowed.")
    else:
        print("Result: " + str(result))
    finally:
        print("Execution completed.")

safe_divide()
```

**Output:**
```
Enter numerator: 10
Enter denominator: 0
Error: Division by zero is not allowed.
Execution completed.
```

### Problem 3: Number Guessing Game

Write a number guessing game with exception handling.

```python
import random

def guess_game():
    target = random.randint(1, 100)
    attempts = 0
    print("Guess the number (1-100)!")

    while True:
        try:
            guess = int(input("Your guess: "))
            attempts += 1
            if guess < 1 or guess > 100:
                print("Out of range! Guess between 1-100.")
                continue
            if guess < target:
                print("Too low!")
            elif guess > target:
                print("Too high!")
            else:
                print("Correct! You guessed it in " + str(attempts) + " attempts.")
                break
        except ValueError:
            print("Invalid input. Enter a number.")

guess_game()
```

**Output:**
```
Guess the number (1-100)!
Your guess: 50
Too low!
Your guess: 75
Too high!
Your guess: 62
Correct! You guessed it in 3 attempts.
```

### Problem 4: File Word Search with Exception Handling

Write a program to search for a word in a file.

```python
def search_word_in_file(filename, word):
    try:
        with open(filename, "r") as f:
            content = f.read()
            count = content.lower().count(word.lower())
            return count
    except FileNotFoundError:
        return -1

filename = input("Enter filename: ")
word = input("Enter word to search: ")
count = search_word_in_file(filename, word)

if count == -1:
    print("File not found.")
else:
    print("The word '" + word + "' appears " + str(count) + " times.")
```

**Output:**
```
Enter filename: sample.txt
Enter word to search: python
The word 'python' appears 0 times.
```

### Problem 5: Raising Exceptions

Write a program that validates age using exception raising.

```python
def set_age(age):
    if age < 0:
        raise ValueError("Age cannot be negative.")
    if age > 150:
        raise ValueError("Age cannot exceed 150.")
    return age

try:
    age_input = int(input("Enter age: "))
    valid_age = set_age(age_input)
    print("Valid age: " + str(valid_age))
except ValueError as e:
    print("Invalid age: " + str(e))
```

**Output:**
```
Enter age: -5
Invalid age: Age cannot be negative.
```
