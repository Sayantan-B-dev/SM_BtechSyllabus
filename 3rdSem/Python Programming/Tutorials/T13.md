### Topic Recap: Exception Handling (try/except/finally), User-Defined Exceptions

**Advanced Exception Handling**:
- Multiple `except` blocks catch different exception types.
- `except (TypeError, ValueError) as e:` — catch multiple types.
- `raise` — manually trigger an exception.
- Creating custom exceptions by subclassing `Exception`.
- Using `assert` for debugging: `assert condition, "message"`

**Best Practices**:
- Catch specific exceptions, not `Exception` broadly.
- Keep `try` blocks minimal.
- Use `finally` for cleanup (closing files, releasing resources).

### Problem 1: Custom Exception Class

Write a custom exception for bank account validation.

```python
class InsufficientBalanceError(Exception):
    def __init__(self, balance, amount):
        self.balance = balance
        self.amount = amount
        super().__init__("Insufficient balance: $" + str(balance) + " < $" + str(amount))

class NegativeAmountError(Exception):
    pass

def withdraw(balance, amount):
    if amount < 0:
        raise NegativeAmountError("Withdrawal amount cannot be negative.")
    if amount > balance:
        raise InsufficientBalanceError(balance, amount)
    return balance - amount

try:
    current_balance = 5000
    withdrawal = float(input("Enter withdrawal amount: "))
    new_balance = withdraw(current_balance, withdrawal)
    print("Withdrawal successful. New balance: $" + str(new_balance))
except NegativeAmountError as e:
    print("Error: " + str(e))
except InsufficientBalanceError as e:
    print("Error: " + str(e))
except ValueError:
    print("Error: Please enter a valid number.")
```

**Output:**
```
Enter withdrawal amount: 7000
Error: Insufficient balance: $5000 < $7000
```

### Problem 2: Nested try/except

Write a program that reads numbers from a file and calculates their sum.

```python
def sum_numbers_from_file(filename):
    total = 0
    try:
        with open(filename, "r") as f:
            for line_num, line in enumerate(f, 1):
                try:
                    num = float(line.strip())
                    total += num
                except ValueError:
                    print("Warning: Line " + str(line_num) + " is not a number, skipping.")
    except FileNotFoundError:
        print("Error: File not found.")
        return None
    return total

total = sum_numbers_from_file("numbers.txt")
if total is not None:
    print("Sum of valid numbers: " + str(total))
```

**Output:**
```
Warning: Line 3 is not a number, skipping.
Warning: Line 5 is not a number, skipping.
Sum of valid numbers: 156.5
```

### Problem 3: Using assert for Validation

Write a program that uses assertions to validate inputs.

```python
def calculate_bmi(weight_kg, height_m):
    assert weight_kg > 0, "Weight must be positive."
    assert height_m > 0, "Height must be positive."
    assert height_m < 3, "Height seems unrealistic (must be in meters)."
    return weight_kg / (height_m ** 2)

try:
    w = float(input("Enter weight (kg): "))
    h = float(input("Enter height (m): "))
    bmi = calculate_bmi(w, h)
    print("BMI: " + str(round(bmi, 2)))
    if bmi < 18.5:
        print("Category: Underweight")
    elif bmi < 25:
        print("Category: Normal")
    elif bmi < 30:
        print("Category: Overweight")
    else:
        print("Category: Obese")
except AssertionError as e:
    print("Assertion error: " + str(e))
except ValueError:
    print("Please enter valid numeric values.")
```

**Output:**
```
Enter weight (kg): 70
Enter height (m): 1.75
BMI: 22.86
Category: Normal
```

### Problem 4: Exception Chaining

Write a program that catches one exception and raises another.

```python
class DataProcessingError(Exception):
    pass

def parse_data(text):
    try:
        return int(text)
    except ValueError as original_error:
        raise DataProcessingError("Failed to parse data.") from original_error

try:
    value = parse_data("not_a_number")
except DataProcessingError as e:
    print("Processing error: " + str(e))
    print("Original cause: " + str(e.__cause__))
```

**Output:**
```
Processing error: Failed to parse data.
Original cause: invalid literal for int() with base 10: 'not_a_number'
```

### Problem 5: Context Manager for File Handling

Write a custom context manager for file operations.

```python
class FileManager:
    def __init__(self, filename, mode):
        self.filename = filename
        self.mode = mode
        self.file = None

    def __enter__(self):
        self.file = open(self.filename, self.mode)
        return self.file

    def __exit__(self, exc_type, exc_val, exc_tb):
        if self.file:
            self.file.close()
        if exc_type:
            print("An exception occurred: " + str(exc_val))
        return True  # suppress exception

with FileManager("test.txt", "w") as f:
    f.write("Hello from context manager!")

with FileManager("test.txt", "r") as f:
    print(f.read())
```

**Output:**
```
Hello from context manager!
```
