### Topic Recap: Pipelining Concepts, Throughput, Speedup

**Pipeline Basics**:
- A technique where multiple instructions overlap in execution.
- **Pipeline stages**: IF (Fetch), ID (Decode), EX (Execute), MEM (Memory), WB (Write Back).
- **Throughput**: Number of instructions completed per unit time.
- **Speedup**: Performance gain from pipelining = Depth of pipeline (ideal case).
- **Pipeline depth**: Number of stages.

### Problem 1: Non-Pipelined vs Pipelined Execution

A non-pipelined processor takes 5 ns per instruction. A 5-stage pipelined processor has each stage taking 1 ns. Compare the time to execute 100 instructions.

**Solution**:

Non-pipelined:
- Time = 100 x 5 ns = 500 ns

Pipelined:
- First instruction: 5 x 1 ns = 5 ns
- After pipeline is full, one instruction completes every 1 ns
- Remaining 99 instructions: 99 x 1 ns = 99 ns
- Total time = 5 + 99 = 104 ns

Speedup = 500 / 104 = 4.8x

Ideal speedup = 5x (pipeline depth), but achieved 4.8x due to startup/fill latency.

### Problem 2: Pipeline Throughput Calculation

A 6-stage pipeline runs at 2 GHz. Calculate the maximum throughput.

**Solution**:

Clock frequency = 2 GHz = 2 x 10^9 Hz

In an ideal pipeline, one instruction completes per clock cycle.

Maximum throughput = 2 x 10^9 instructions per second = 2000 MIPS (Million Instructions Per Second)

If the pipeline has stalls due to hazards, the actual throughput will be lower.

### Problem 3: Pipeline Speedup Formula

Derive and calculate speedup for a k-stage pipeline with n instructions.

**Solution**:

Non-pipelined time = n x t (where t is the time per instruction in a non-pipelined processor, assuming each stage takes same time)

Pipelined time = k x t + (n - 1) x t = (k + n - 1) x t

Speedup = (n x t) / ((k + n - 1) x t) = n / (k + n - 1)

For large n: Speedup approaches k (pipeline depth)

Example: k=5, n=1000
Speedup = 1000 / (5 + 1000 - 1) = 1000 / 1004 = 0.996 x 5 = 4.98

For n=10: Speedup = 10 / (5 + 10 - 1) = 10/14 = 0.714 x 5 = 3.57

### Problem 4: Pipeline with Stalls

A 5-stage pipeline has a 20% chance of a branch instruction that causes 3 stall cycles. Calculate the effective CPI (Cycles Per Instruction) and throughput at 1 GHz.

**Solution**:

Ideal CPI = 1 (one instruction per cycle)

Branch instructions: 20%, each causing 3 stalls
Other instructions: 80%, no stalls

Average CPI = 1 + (0.20 x 3) = 1 + 0.6 = 1.6

Throughput at 1 GHz = 1 GHz / 1.6 = 625 MIPS

Without branches, throughput would be 1000 MIPS. Branches reduce throughput by 37.5%.

### Problem 5: Dependencies and Forwarding

Identify data dependencies in this instruction sequence and determine how many stalls are needed with and without forwarding.

```
I1: ADD R1, R2, R3   ; R1 = R2 + R3
I2: SUB R4, R1, R5   ; R4 = R1 - R5 (reads R1 from I1)
I3: AND R6, R1, R7   ; R6 = R1 AND R7 (reads R1 from I1)
I4: OR R8, R4, R9    ; R8 = R4 OR R9 (reads R4 from I2)
```

**Solution**:

Dependencies:
- I2 reads R1 written by I1 (RAW dependency)
- I3 reads R1 written by I1 (RAW dependency)
- I4 reads R4 written by I2 (RAW dependency)

Without forwarding:
- I1 -> I2: 2 stall cycles (EX -> WB then ID reads)
- I1 -> I3: 2 stall cycles
- I2 -> I4: 2 stall cycles
Total cycles: 4 (instructions) + 6 (stalls) = 10 cycles

With forwarding:
- I1 -> I2: 0 stalls (forward from EX stage of I1 to EX stage of I2)
- I1 -> I3: 0 stalls
- I2 -> I4: 1 stall (forward from MEM stage of I2 to EX stage of I4)
Wait, actually with full forwarding:
- I2's result is available at the end of I2's EX stage. I4 needs it at the start of I4's EX stage.
- If I2's EX is in cycle 3, I3's ID is in cycle 4, I4's ID is in cycle 5, I4's EX is in cycle 6.
- So I2 (EX in cycle 3) -> I4 (EX in cycle 6): 2 cycles gap. With forwarding, can resolve with 0 stalls.

Actually with forwarding, all RAW hazards can be resolved without stalling if the dependent instruction is at least 2 cycles away. For back-to-back RAW: 1 stall may be needed (EX of I1 -> EX of I2, need the result from EX of I1 but I2 starts EX in same cycle).

With forwarding: 1 stall for back-to-back ALU RAW. So I1->I2: 1 stall, I2->I4: 0 stalls (one instruction apart).
