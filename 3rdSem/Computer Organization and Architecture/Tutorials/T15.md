### Problem 1: CPU Performance Equation (Module 1)

A processor runs at 2.5 GHz. A program has 1 million instructions with an average CPI of 1.8. Calculate execution time.

**Solution**:

Execution time = (Instruction count x CPI) / Clock frequency
= (10^6 x 1.8) / (2.5 x 10^9)
= 1.8 x 10^6 / 2.5 x 10^9
= 0.72 x 10^-3
= 0.72 ms

If we optimize the code to reduce instruction count by 20% but CPI increases to 2.1, calculate the new time:

New instruction count = 0.8 x 10^6
New execution time = (0.8 x 10^6 x 2.1) / (2.5 x 10^9)
= 1.68 x 10^6 / 2.5 x 10^9
= 0.672 ms

Speedup = 0.72 / 0.672 = 1.07x (7% improvement)

### Problem 2: Number Conversion and Arithmetic (Module 2)

Add -35 and +22 in 8-bit 2's complement. Check for overflow.

**Solution**:

+35 = 00100011
-35 = 11011101 (2's complement)

+22 = 00010110

```
  11011101 (-35)
+ 00010110 (+22)
-----------
  11110011
```

Check: 11110011 -> subtract 1: 11110010 -> invert: 00001101 = 13
So result = -13. Correct (since -35 + 22 = -13).

Overflow check: Two's complement overflow occurs when sign of inputs are same but sign of result differs.
Sign of A = 1, Sign of B = 0 (different signs) -> No overflow possible.

### Problem 3: Memory Hierarchy Optimization (Module 3 + 5)

A program has an L1 miss rate of 8%, L2 miss rate of 30% (of L1 misses). L1 hit = 1 cycle, L2 hit = 10 cycles, main memory = 100 cycles. Clock = 3 GHz.

(a) Calculate average memory access time (AMAT).
(b) If we double L1 size, L1 miss rate drops to 5% but L1 hit time becomes 2 cycles. Is this beneficial?

**Solution**:

(a) AMAT = L1_hit + L1_miss_rate x (L2_hit + L2_miss_rate x MM_hit)
= 1 + 0.08 x (10 + 0.30 x 100)
= 1 + 0.08 x (10 + 30)
= 1 + 0.08 x 40
= 1 + 3.2
= 4.2 cycles

At 3 GHz (0.33 ns/cycle): 4.2 x 0.33 = 1.39 ns

(b) New AMAT = 2 + 0.05 x (10 + 0.30 x 100)
= 2 + 0.05 x 40
= 2 + 2.0
= 4.0 cycles

At 3 GHz: 4.0 x 0.33 = 1.33 ns

Improvement: 1.39 -> 1.33 ns (4.8% improvement). Doubling L1 is beneficial.

### Problem 4: Pipeline Speedup with Hazards (Module 4)

A 5-stage pipeline has:
- 15% loads (1 stall cycle each)
- 10% branches (2 stall cycles each)
- 5% stores (no stalls)
- 70% ALU instructions (no stalls)

Calculate the effective CPI and speedup over a non-pipelined processor where each instruction takes 5 cycles.

**Solution**:

Stall cycles per instruction:
- Loads: 0.15 x 1 = 0.15
- Branches: 0.10 x 2 = 0.20
- Stores: 0.05 x 0 = 0.00
- ALU: 0.70 x 0 = 0.00

Total stalls per instruction = 0.15 + 0.20 = 0.35

Effective CPI = 1 (ideal) + 0.35 = 1.35

Non-pipelined CPI = 5
Pipelined CPI = 1.35

Speedup = (5 x 1) / (1 x 1.35) = 5 / 1.35 = 3.7x

Ideal speedup = 5x. Hazards reduce it to 3.7x.

### Problem 5: Cache Performance (Module 5)

A 16 KB direct-mapped cache with 16-byte blocks. Access sequence (byte addresses): 0, 16, 32, 48, 64, 80, 96, 112, 128, 144, 160, 176, 0, 16, 32, 48.

Calculate miss rate and identify miss types.

**Solution**:

Block size = 16 bytes, offset = 4 bits
Number of blocks = 16 KB / 16 B = 1024 blocks, index = 10 bits

Addresses in block form:
- 0: Block 0, Index 0
- 16: Block 1, Index 1
- 32: Block 2, Index 2
- 48: Block 3, Index 3
- 64: Block 4, Index 4
- 80: Block 5, Index 5
- 96: Block 6, Index 6
- 112: Block 7, Index 7
- 128: Block 8, Index 8
- 144: Block 9, Index 9
- 160: Block 10, Index 10
- 176: Block 11, Index 11
- 0: Block 0, Index 0
- 16: Block 1, Index 1
- 32: Block 2, Index 2
- 48: Block 3, Index 3

First 12 accesses: All compulsory misses (12 misses)
Next 4 accesses: Cache hits (blocks already in cache from first pass)

Miss rate = 12/16 = 75%

All misses are compulsory (cold misses). No conflict or capacity misses because 1024 blocks >> 4 blocks needed for this working set.

### Problem 6: I/O Performance (Module 3)

A disk transfers data at 200 MB/s using DMA. The DMA controller requires 2 microseconds to setup and 1 microsecond for completion interrupt. Compare CPU overhead for transferring 4 KB vs 1 MB data.

**Solution**:

**4 KB transfer**:
- DMA setup: 2 us
- DMA transfer: 4 KB / 200 MB/s = 4096 / (200 x 10^6) = 20.48 us
- Completion interrupt: 1 us
- CPU overhead: 2 + 1 = 3 us (DMA transfer is hardware, no CPU involved)

**1 MB transfer**:
- DMA setup: 2 us
- DMA transfer: 1 MB / 200 MB/s = 10^6 / (200 x 10^6) = 5 ms
- Completion interrupt: 1 us
- CPU overhead: 2 + 1 = 3 us (same setup!)

For interrupt-driven I/O at 1 byte per interrupt (2 us ISR):
- 4 KB: 4096 x 2 us = 8.2 ms CPU time
- 1 MB: 1,048,576 x 2 us = 2.1 seconds CPU time

DMA is 2700x more efficient for 4 KB and 700,000x more efficient for 1 MB.
