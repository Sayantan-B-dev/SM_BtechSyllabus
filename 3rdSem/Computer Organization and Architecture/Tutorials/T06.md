### Topic Recap: Division Algorithms, Floating-Point Arithmetic

**Division Methods**:
- **Restoring Division**: Subtract divisor, if result negative restore previous remainder.
- **Non-Restoring Division**: More efficient. If remainder negative, add divisor instead of restoring.

**IEEE 754 Floating Point**:
- Single precision (32-bit): 1 sign, 8 exponent (bias 127), 23 mantissa
- Double precision (64-bit): 1 sign, 11 exponent (bias 1023), 52 mantissa
- Special values: NaN, Infinity, Denormalized numbers

### Problem 2: Non-Restoring Division

Compare restoring vs non-restoring division.

**Restoring**: After subtraction, if negative, add back divisor (restore). Two operations per step.

**Non-Restoring**: If remainder negative, shift left and ADD divisor (instead of subtract then restore). One operation per step.

For 7 / 3:
- Restoring: 4 iterations, up to 8 add/sub operations
- Non-restoring: 4 iterations, exactly 4 add/sub operations

The non-restoring method saves about half the operations.

### Problem 3: IEEE 754 Addition

Add 5.75 and 2.125 in IEEE 754 single precision.

**Solution**:

5.75 = 101.11 = 1.0111 x 2^2
2.125 = 10.001 = 1.0001 x 2^1

Step 1: Align exponents (match to larger exponent = 2)
- 1.0001 x 2^1 -> 0.10001 x 2^2

Step 2: Add mantissas
- 1.0111 + 0.10001 = 1.11111

Step 3: Result = 1.11111 x 2^2

Step 4: Normalize (already normalized)
= 111.111 in binary = 7.875 in decimal

Step 5: Encode
- Sign: 0
- Exponent: 2 + 127 = 129 = 10000001
- Mantissa: 11111000000000000000000

IEEE 754: 0 10000001 11111000000000000000000

### Problem 4: Floating Point Multiplication

Multiply 5.75 x 2.125 using IEEE 754.

**Solution**:

5.75 = 1.0111 x 2^2
2.125 = 1.0001 x 2^1

Step 1: Add exponents: 2 + 1 = 3
Step 2: Multiply mantissas:
  1.0111 x 1.0001 = 1.0111 x (1 + 0.0001)
  = 1.0111 + 0.00010111 = 1.10000111

Step 3: Result mantissa = 1.10000111, exponent = 3

Step 4: Normalize (already normalized)
= 1100.00111 = 12.21875

Check: 5.75 x 2.125 = 12.21875. Correct!
