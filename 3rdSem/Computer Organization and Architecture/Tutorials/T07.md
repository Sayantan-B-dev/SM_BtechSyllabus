### Topic Recap: Semiconductor Memory, SRAM/DRAM/ROM

**Memory Types**:
- **SRAM (Static RAM)**: Uses flip-flops. Fast, low density, high power. Used for cache.
- **DRAM (Dynamic RAM)**: Uses capacitors. Slow, high density, low power. Needs periodic refresh. Used for main memory.
- **ROM (Read-Only Memory)**: Non-volatile. Types: Mask ROM, PROM, EPROM, EEPROM, Flash.
- **Memory Organization**: N words x M bits per word.

**Key Parameters**:
- Capacity = Number of words x Word size
- Access time, Cycle time
- Volatility, Power consumption, Cost per bit

### Problem 1: Memory Capacity Calculation

How many bits are needed to address a 64K x 16 memory?

**Solution**:

64K = 64 x 1024 = 65536 words
Number of address bits = log2(65536) = 16

Total bits = 65536 x 16 = 1,048,576 bits = 1 Mbit

So 16 address lines are needed.

### Problem 2: Memory Chip Organization

Design a 4K x 8 memory using 1K x 8 chips.

**Solution**:

Each chip: 1K x 8 (10 address lines, 8 data lines)
Need: 4K x 8 (12 address lines, 8 data lines)

Number of chips = 4K / 1K = 4 chips

Address decoding:
- 10 bits (A0-A9) go to each chip (chip address)
- 2 bits (A10-A11) go to a 2-to-4 decoder to select one of 4 chips

```
A10 A11 -> Decoder -> Chip Selects:
  0   0   -> CS0 (chip 0: addresses 0x000-0x3FF)
  0   1   -> CS1 (chip 1: addresses 0x400-0x7FF)
  1   0   -> CS2 (chip 2: addresses 0x800-0xBFF)
  1   1   -> CS3 (chip 3: addresses 0xC00-0xFFF)
```

### Problem 3: SRAM vs DRAM Comparison

Compare SRAM and DRAM across key parameters.

| Parameter | SRAM | DRAM |
|-----------|------|------|
| Storage element | Flip-flop (6 transistors) | Capacitor + transistor |
| Speed | Fast (1-10 ns) | Slower (10-100 ns) |
| Density | Low (large cell size) | High (small cell size) |
| Power | Higher (continuous current) | Lower (but needs refresh) |
| Volatility | Volatile | Volatile |
| Refresh needed | No | Yes (every 64 ms) |
| Cost per bit | Higher | Lower |
| Application | Cache memory | Main memory |

### Problem 4: DRAM Refresh Calculation

A 4 Mbit DRAM chip is organized as 4096 rows. The recommended refresh interval is 64 ms. Calculate the refresh frequency.

**Solution**:

Number of rows = 4096
Refresh interval = 64 ms

Each row must be refreshed every 64 ms.
Time between consecutive row refreshes = 64 ms / 4096 = 15.625 microseconds

Refresh frequency = 1 / 15.625 us = 64 kHz

If the DRAM controller refreshes one row per cycle, it needs a timer that triggers every 15.625 microseconds.

### Problem 6: Memory Hierarchy

Calculate the average memory access time given:
- L1 Cache: 1 ns, 95% hit rate
- L2 Cache: 5 ns, 85% hit rate (of misses)
- Main Memory: 50 ns

**Solution**:

Average access time = T_L1 + (1 - H_L1) x T_L2 + (1 - H_L1) x (1 - H_L2) x T_MM

H_L1 = 0.95 (hit rate in L1)
H_L2 = 0.85 (hit rate in L2, given a miss in L1)

Average time = 1 + 0.05 x 5 + 0.05 x 0.15 x 50
= 1 + 0.25 + 0.375
= 1.625 ns

Without cache hierarchy, every access would take 50 ns. The cache hierarchy reduces the average access time dramatically.
