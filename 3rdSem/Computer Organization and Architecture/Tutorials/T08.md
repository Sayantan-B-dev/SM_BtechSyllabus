### Topic Recap: I/O Transfers (Programmed, Interrupt-Driven, DMA)

**I/O Transfer Methods**:
- **Programmed I/O**: CPU polls devices. Wastes CPU time.
- **Interrupt-Driven I/O**: Device interrupts CPU when ready. More efficient.
- **DMA (Direct Memory Access)**: DMA controller transfers data directly between device and memory. CPU not involved in data transfer.

**I/O Interface**: Connects I/O devices to the system bus. Contains data, status, and control registers.

### Problem 1: Programmed I/O Timing

A printer can print 100 characters per second. The CPU polls the printer every 500 microseconds. Each poll takes 10 microseconds. What percentage of CPU time is spent polling?

**Solution**:

Printer speed: 100 chars/sec -> one char every 10 ms

The CPU polls every 500 microseconds = 0.5 ms

Number of polls per character = 10 ms / 0.5 ms = 20 polls

Total polling time per character = 20 x 10 us = 200 microseconds

Time per character = 10 ms = 10000 microseconds

CPU utilization for polling = 200 / 10000 = 2%

While 2% seems small, if there are many devices being polled, the overhead accumulates significantly.

### Problem 2: Interrupt-Driven I/O

In an interrupt-driven I/O system, the keyboard generates an interrupt for every keystroke. The interrupt service routine takes 50 microseconds. If the user types at 10 characters per second, what is the CPU overhead?

**Solution**:

Typing rate: 10 chars/sec -> one char every 100 ms

Interrupt service time per character = 50 microseconds

CPU overhead = 50 / 100000 = 0.05%

Compare with programmed I/O polling at 500 us intervals:
- Polls per character = 100 ms / 0.5 ms = 200
- Polling time per character = 200 x 10 us = 2000 us
- Overhead = 2000 / 100000 = 2%

Interrupt-driven I/O is 40x more efficient in this example.

### Problem 3: DMA Transfer

A DMA controller transfers data from a disk to memory at 100 MB/s. The disk transfer size is 4 KB per sector. Calculate the time to transfer one sector.

**Solution**:

Transfer rate = 100 MB/s = 100 x 10^6 bytes/sec

Sector size = 4 KB = 4096 bytes

Transfer time = 4096 / (100 x 10^6) = 40.96 microseconds

During this time, the DMA controller holds the bus. The CPU is blocked from accessing memory for 41 microseconds.

### Problem 4: DMA vs Interrupt for Block Transfer

Compare the CPU overhead for transferring a 1 MB file from disk to memory using:
(a) Interrupt-driven I/O (1 byte per interrupt, 2 us ISR)
(b) DMA (burst transfer, 10 us setup + 5 us completion interrupt)

**Solution**:

(a) Interrupt-driven:
- Number of interrupts = 1 MB / 1 byte = 1,048,576
- Total ISR time = 1,048,576 x 2 us = 2,097,152 us = 2.1 seconds
- CPU is fully occupied during this time

(b) DMA:
- Setup time = 10 us
- Completion interrupt = 5 us
- Total CPU time = 15 us
- DMA controller does the actual data transfer

DMA is dramatically more efficient for block transfers.

### Problem 5: Daisy Chaining Interrupts

Three devices (A, B, C) are connected via daisy chain with priorities A > B > C. Device B and C simultaneously request an interrupt. Describe what happens.

**Solution**:

1. Both B and C assert their interrupt request lines.
2. The CPU (or interrupt controller) acknowledges the interrupt.
3. The acknowledgment signal travels through the daisy chain.
4. Device A does not need service, so it passes the acknowledgment to B.
5. Device B recognizes the acknowledgment, takes the bus, and sends its interrupt vector. It blocks the acknowledgment from reaching C.
6. CPU services B. B's ISR runs.
7. After B's ISR completes, the interrupt cycle repeats.
8. The acknowledgment goes through A (skip), then B (no longer requesting), then reaches C.
9. C is acknowledged and serviced.

Priority is determined by position in the chain. Device A has the highest priority (closest to CPU).
