### Topic Recap: Memory Interleaving, Hierarchical Memory Organization, Cache Memory Basics

**Memory Interleaving**: Splitting memory into multiple banks accessed in parallel. High-order vs low-order interleaving.

**Memory Hierarchy**: Registers -> L1 Cache -> L2 Cache -> L3 Cache -> Main Memory -> Disk (increasing capacity, decreasing speed).

**Cache Memory Basics**:
- Cache hit: Data found in cache (fast).
- Cache miss: Data not in cache (fetch from slower memory).
- Hit rate = Hits / Total accesses.
- Average access time = Hit time + Miss rate x Miss penalty.

### Problem 1: Low-Order vs High-Order Interleaving

A 64 KB memory is organized as 4 banks of 16 KB each. Show how address 0x1234 maps to a bank in both schemes.

**Solution**:

Address = 0x1234 = 0001 0010 0011 0100 (16 bits)

**High-Order Interleaving**:
- Upper 2 bits select bank: 00 -> Bank 0
- Remaining 14 bits select word within bank

```
0x1234 = 00 01001000110100
Bank = 00 (Bank 0)
Word = 0x1234 (within bank)
```

Consecutive addresses go to the same bank. Not good for sequential access.

**Low-Order Interleaving**:
- Lower 2 bits select bank: 00 -> Bank 0
- Remaining 14 bits select word within bank

```
0x1234 = 01001000110100 00
Bank = 00 (Bank 0)
Word = 0x048D
```

Consecutive addresses go to different banks. Good for sequential access (pipelining memory requests).

### Problem 2: Memory Interleaving Performance

A memory system has 4 banks with 50 ns access time per bank. Without interleaving, sequential reads take 50 ns per word. With interleaving, how long does it take to read 4 consecutive words?

**Solution**:

Without interleaving: 4 x 50 ns = 200 ns

With low-order interleaving:
- Bank 0: Word 0 (50 ns)
- Bank 1: Word 1 (starts after bank 0 finishes, or can start in parallel if banks are independent)

If banks are truly independent and can be accessed in parallel:
- All 4 banks start simultaneously
- Time = 50 ns (all words read in parallel)

If banks share some resources (e.g., data bus):
- Words can be pipelined: start bank i+1 while bank i is still being read
- With pipelining: 50 ns for first word + 3 x (stagger delay) 
- If stagger = 10 ns: 50 + 30 = 80 ns

Interleaving improves bandwidth by 2.5x (200/80) in the pipelined case.

### Problem 3: Cache Structure

A 64 KB direct-mapped cache has 32-byte blocks. Calculate:
(a) Number of blocks
(b) Number of bits for index, offset, and tag for a 32-bit address

**Solution**:

(a) Number of blocks = Cache size / Block size = 64 KB / 32 B = 65536 / 32 = 2048 blocks

(b) Address breakdown (32-bit):
- Offset: log2(32) = 5 bits (which byte within the block)
- Index: log2(2048) = 11 bits (which cache line)
- Tag: 32 - 5 - 11 = 16 bits

```
| Tag (16 bits) | Index (11 bits) | Offset (5 bits) |
```

Cache entry: Tag (16 bits) + Data (32 bytes = 256 bits) + Valid bit (1 bit) = 273 bits per block
Total cache size (including tags): 2048 x 273 = 559,104 bits = 68.25 KB

### Problem 4: Cache Hit/Miss Calculation

A program accesses memory addresses: 0, 4, 8, 12, 16, 20, 24, 28, 32, 36 (all in bytes). Cache is 32-byte direct-mapped with 8-byte blocks. Calculate hits and misses.

**Solution**:

Block size = 8 bytes, so offset = 3 bits
Cache size = 32 bytes, blocks = 32/8 = 4, so index = 2 bits

Address breakdown (using only needed bits):

| Address | Binary (partial) | Index | Tag | Block | Hit/Miss |
|---------|-----------------|-------|-----|-------|----------|
| 0 | 00000 | 00 | 0 | Block 0 | Miss (compulsory) |
| 4 | 00100 | 00 | 0 | Block 0 | Hit |
| 8 | 01000 | 01 | 0 | Block 1 | Miss |
| 12 | 01100 | 01 | 0 | Block 1 | Hit |
| 16 | 10000 | 00 | 1 | Block 2 | Miss (new tag) |
| 20 | 10100 | 01 | 1 | Block 3 | Miss (new tag) |
| 24 | 11000 | 00 | 1 | Block 2 | Hit |
| 28 | 11100 | 01 | 1 | Block 3 | Hit |
| 32 | 00000 | 00 | 0 | Block 0 | Miss (was evicted by addr 16) |
| 36 | 00100 | 00 | 0 | Block 0 | Hit |

Hits: 4, Misses: 5 (Compulsory: 4, Conflict: 1 at addr 32)
Hit rate: 4/10 = 40%

Notice: Even though the sequence is sequential, the 4-block cache experiences conflict misses because the pattern exceeds the cache capacity for a given index.

### Problem 5: Inclusion Policy in Multi-Level Cache

An L1 cache (32 KB) and L2 cache (256 KB). Explain the inclusion property for:
(a) Inclusive cache: L1 data is always a subset of L2.
(b) Exclusive cache: L1 and L2 have no overlapping data.
(c) Non-inclusive non-exclusive (NINE): No constraints.

**Solution**:

(a) **Inclusive**: When L1 evicts a block, L2 still has it. Simplifies coherence but wastes L2 space.
(b) **Exclusive**: When L1 fetches from L2, it swaps. Maximizes effective capacity but complex.
(c) **NINE**: Most flexible. L2 may or may not contain L1 data. Simple but coherence handling is harder.

Intel uses inclusive L2 caches in many designs. AMD has used exclusive and NINE policies.
