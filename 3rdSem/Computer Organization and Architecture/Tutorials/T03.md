## Notes

### Topic Recap: Signed Number Representations

**Number Representations**:
- **Unsigned**: All bits represent magnitude (0 to 2^n - 1).
- **Signed Magnitude**: MSB = sign (0=positive, 1=negative), rest = magnitude.
- **1's Complement**: Negative = bitwise NOT of positive.
- **2's Complement**: Negative = 1's complement + 1. Most common representation.
- **Fixed Point**: Integer part + fractional part, binary point at fixed position.
- **Floating Point**: Representation with mantissa and exponent (IEEE 754 standard).

### Problem 1: Represent in Different Formats

Represent +25 and -25 in 8-bit signed magnitude, 1's complement, and 2's complement.

**Solution**:

+25 in binary: 00011001

| Format | +25 | -25 |
|--------|-----|-----|
| Signed Magnitude | 00011001 | 10011001 |
| 1's Complement | 00011001 | 11100110 |
| 2's Complement | 00011001 | 11100111 |

### Problem 2: Range Calculation

Calculate the range of values for each 8-bit representation.

**Solution**:

| Representation | Minimum | Maximum | Range |
|---------------|---------|---------|-------|
| Unsigned | 0 | 255 | 0 to 255 |
| Signed Magnitude | -127 | +127 | -(2^7-1) to +(2^7-1) |
| 1's Complement | -127 | +127 | -(2^7-1) to +(2^7-1) |
| 2's Complement | -128 | +127 | -2^7 to +(2^7-1) |

Note: 2's complement has an asymmetric range because there is only one representation for zero.

### Problem 3: Convert Decimal to 2's Complement

Convert -47 to 8-bit 2's complement.

**Solution**:

Step 1: +47 in binary = 00101111

Step 2: 1's complement = 11010000

Step 3: Add 1: 11010000 + 1 = 11010001

Therefore, -47 in 8-bit 2's complement = 11010001

### Problem 4: Convert 2's Complement to Decimal

Convert 11001010 (8-bit 2's complement) to decimal.

**Solution**:

MSB = 1, so the number is negative.

Step 1: Subtract 1: 11001010 - 1 = 11001001

Step 2: 1's complement (invert): 00110110

Step 3: Convert to decimal: 32 + 16 + 4 + 2 = 54

Therefore, 11001010 = -54

### Problem 5: Fixed Point Representation

Represent 13.375 in fixed-point binary (8 bits: 4 integer, 4 fractional).

**Solution**:

Integer part 13: 1101

Fractional part 0.375:
- 0.375 x 2 = 0.75 -> 0
- 0.75 x 2 = 1.5 -> 1
- 0.5 x 2 = 1.0 -> 1
- 0.0 -> stop

So 0.375 = 0110 binary (rounded to 4 bits)

13.375 = 1101.0110 in fixed point

### Problem 6: IEEE 754 Single Precision (32-bit)

Represent 5.75 in IEEE 754 single precision.

**Solution**:

Step 1: Convert to binary: 5 = 101, 0.75 = 11, so 5.75 = 101.11

Step 2: Normalize: 1.0111 x 2^2

Step 3: Sign bit = 0 (positive)

Step 4: Exponent = 2 + 127 = 129 = 10000001 binary

Step 5: Mantissa = 01110000000000000000000 (fractional part after the leading 1)

Result: 0 | 10000001 | 01110000000000000000000

In hex: 0x40B80000
