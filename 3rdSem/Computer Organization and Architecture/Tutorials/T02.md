## Notes

### Topic Recap: CPU Registers, Instruction Execution Cycle, RTL

**CPU Registers**:
- **Program Counter (PC)**: Holds address of next instruction.
- **Instruction Register (IR)**: Holds current instruction being executed.
- **Memory Address Register (MAR)**: Holds address for memory access.
- **Memory Buffer/Data Register (MBR/MDR)**: Holds data read from or written to memory.
- **Accumulator (ACC)**: Stores intermediate results.
- **General Purpose Registers (R0-Rn)**: Temporary storage.

**Instruction Execution Cycle**: Fetch -> Decode -> Execute -> Write Back.

**Register Transfer Language (RTL)**: Notation describing data flow between registers.
- Notation: `[Source] -> Destination`
- Example: `[PC] -> MAR` means "content of PC is transferred to MAR"

### Problem 1: RTL for Instruction Fetch

Write the RTL sequence for the fetch phase of the instruction cycle.

**Solution**:

Step 1: Send PC address to memory
```
[PC] -> MAR
```

Step 2: Request memory read
```
READ -> Control Bus
```

Step 3: Wait for memory (read cycle), then load instruction into IR
```
[MEM[MAR]] -> MBR
[MBR] -> IR
```

Step 4: Increment PC for next instruction
```
[PC] + 1 -> PC
```

Complete RTL sequence:
```
T0: [PC] -> MAR
T1: READ -> MEM, [MEM[MAR]] -> MBR
T2: [MBR] -> IR, [PC] + 1 -> PC
```

### Problem 2: RTL for ADD Instruction

Write the RTL sequence for `ADD R1, R2` (R1 = R1 + R2).

**Solution**:

Assuming the instruction `ADD R1, R2` is already in IR:

```
T0: [R2] -> ALU_INPUT_B
T1: [R1] -> ALU_INPUT_A, ALU performs ADD
T2: [ALU_OUT] -> R1
```

However, this is simplified. In practice, the control unit generates micro-operations:

```
T0: [R1] -> TEMP (temporary register)
T1: [R2] -> ALU input
T2: ALU adds TEMP and ALU input
T3: [ALU output] -> R1
```

The exact number of micro-operations depends on the CPU design.

### Problem 3: RTL for STORE Instruction

Write the RTL for `STORE R1, [1000]` (store R1 content to memory address 1000).

**Solution**:

```
T0: [Address field of IR] -> MAR (address 1000)
T1: [R1] -> MBR
T2: WRITE -> MEM, [MBR] -> MEM[MAR]
```

So the value in R1 is placed on the data bus and written to memory location 1000.

### Problem 5: Memory Read Timing

A CPU has a clock of 2 GHz. Each memory read takes 3 clock cycles. How long does it take to fetch an instruction?

**Solution**:

Clock frequency = 2 GHz = 2 x 10^9 Hz
Clock period = 1 / (2 x 10^9) = 0.5 ns

One memory read = 3 clock cycles = 3 x 0.5 ns = 1.5 ns

The instruction fetch phase takes 1.5 ns (excluding decode and execute).
