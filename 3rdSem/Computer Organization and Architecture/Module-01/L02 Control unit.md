## Notes

### 1. Introduction to the Control Unit

The Control Unit (CU) is one of the three main components of the CPU (along with the ALU and registers). Its primary job is to coordinate and direct all operations of the computer. It acts as the "traffic controller" of the CPU.

**Key responsibilities:**
- Fetch instructions from main memory
- Decode instructions to determine the operation to be performed
- Generate control signals that direct the ALU, memory, I/O devices, and registers
- Manage the sequential execution of instructions (the instruction cycle)
- Respond to interrupts and exceptional conditions
- Provide timing signals to synchronize operations

### 2. Functions of the Control Unit

The control unit performs the following major functions:

#### 2.1 Instruction Fetching
- Sends the contents of the Program Counter (PC) to the Memory Address Register (MAR).
- Activates a memory read operation (MemRead signal) via the control bus.
- The memory returns the instruction, which is placed in the Memory Buffer Register (MBR).
- The instruction is moved from MBR to the Instruction Register (IR).
- The PC is incremented to point to the next instruction.

#### 2.2 Instruction Decoding
- Examines the opcode field of the instruction in the IR.
- Determines which operation (ADD, LOAD, STORE, JUMP, etc.) is to be performed.
- Determines the addressing mode used (direct, indirect, immediate, etc.).
- Identifies which registers or memory locations are involved as operands.

#### 2.3 Control Signal Generation
- The CU generates a sequence of **control signals** that activate the appropriate hardware paths.
- These signals open/close gates (tri-state buffers), enable/disable registers, trigger ALU operations, initiate memory reads/writes, and manage I/O.
- Each control signal is asserted (set to 1) at specific clock cycles within the instruction cycle.

#### 2.4 Sequencing
- The CU ensures that micro-operations are performed in the correct order.
- It uses a **sequence counter** or a **state machine** to track which step of an instruction is currently being executed.
- For example, the fetch phase is always done before the execute phase; the ALU operation is done after operands are ready.

#### 2.5 Interrupt Handling
- The CU checks for interrupts at the end of each instruction cycle.
- If an interrupt is pending and interrupts are enabled, the CU:
  1. Saves the current PC value (return address) onto the stack.
  2. Loads the address of the interrupt service routine (ISR) into the PC.
  3. Disables further interrupts (or masks them).
  4. Executes the ISR.
  5. On return from interrupt (IRET), restores the saved PC.

### 3. Control Signals

Control signals are binary signals (0 or 1) that enable or disable specific hardware components. They are generated by the control unit based on:
- The current instruction (opcode in IR).
- The current step of the instruction cycle (T-state).
- Condition flags (Zero flag, Carry flag, etc.).

**Examples of control signals:**

| Control Signal | Function | Asserted When |
|----------------|----------|---------------|
| PCtoMAR | Enable PC output onto address bus | Every fetch cycle |
| MemRead | Initiate memory read operation | Fetch, LOAD, etc. |
| MARtoMBR | Read data from memory into MBR | After MemRead completes |
| MBRtoIR | Transfer MBR contents to IR | End of fetch cycle |
| PCinc | Increment the Program Counter | After instruction is fetched |
| IRtoControl | Decode IR contents to generate signals | Decode cycle |
| ALUop | Select ALU operation (ADD, SUB, etc.) | Execute cycle |
| RegWrite | Enable writing to destination register | End of execute cycle |
| MemWrite | Initiate memory write operation | STORE instruction |
| IntAck | Acknowledge interrupt request | Interrupt cycle |

### 4. Hardwired Control Unit

In a hardwired control unit, the control signals are generated by combinational logic circuits (gates, flip-flops, decoders, etc.).

**How it works:**
- The opcode from the IR is decoded using a decoder.
- A **sequence counter** or a **ring counter** tracks the current cycle step.
- Combinational logic uses the decoded opcode, current step, and condition flags to produce control signals.
- The control logic is physically wired into the circuit.

**Block diagram of hardwired control:**

```
+-----------+     +-----------+     +----------------+
|   IR      |---->| Opcode    |---->| Control Logic  |
| (Opcode)  |     | Decoder   |     | (Combinational |
+-----------+     +-----------+     |  Circuits)     |
                                    |                |
+-----------+     +-----------+     |  +----------+  |
| Clock     |---->| Sequence  |---->|  | AND/OR   |  |
|           |     | Counter   |     |  | Gates    |  |
+-----------+     | (T-states)|     |  +----------+  |
                  +-----------+     +--------+-------+
                                             |
                                             v
                                     +-------+-------+
                                     | Control Signals|
                                     | (to ALU, Mem, |
                                     |  Registers,   |
                                     |  I/O, etc.)   |
                                     +---------------+
```

**Advantages:**
- Very fast (control signals are generated by hardware gates with minimal delay).
- Suitable for RISC processors where instruction sets are simple and fixed.
- Good for high-performance implementations.

**Disadvantages:**
- Complex to design and debug (especially for large instruction sets).
- Inflexible: adding a new instruction requires redesigning the hardware.
- Not suitable for CISC processors with many complex instructions.
- More expensive to develop (longer design time).

### 5. Microprogrammed Control Unit

In a microprogrammed control unit, control signals are generated by executing **microinstructions** stored in a special memory called **control store** (usually ROM or PLA).

**Key concepts:**
- Each machine instruction (ADD, LOAD, etc.) is implemented as a sequence of **microinstructions**.
- A microinstruction contains a set of control bits that activate specific hardware paths.
- The **microprogram** is the collection of all microinstruction sequences for all machine instructions.
- A **microprogram counter (microPC)** keeps track of the current microinstruction.
- **Microprogram sequencing logic** determines the next microinstruction address (sequentially, branch, or from the opcode).

**How it works:**
1. The opcode from the IR is mapped to the starting address of the corresponding microprogram routine.
2. The microPC loads this starting address.
3. Microinstructions are fetched from control store and executed one by one.
4. Each microinstruction generates control signals for the current cycle.
5. The last microinstruction in the routine either branches to the next instruction's microcode or returns to the fetch routine.

**Block diagram of microprogrammed control:**

```
+-----------+     +-----------+     +-------------------+
|   IR      |---->| Mapping   |---->| Control Store     |
| (Opcode)  |     | Logic     |     | (ROM containing   |
+-----------+     +-----------+     |  microinstructions)|
                                    +--------+----------+
                                             |
                        +--------------------+
                        |                    |
                        v                    v
                 +------+------+     +------+------+
                 | MicroPC    |     | Micro-       |
                 | (next addr)|     | Instruction  |
                 +------+------+     | Register (uIR)|
                        |            +------+-------+
                        v                   |
                 +------+------+            |
                 | Sequencing  |            v
                 | Logic       |     +------+-------+
                 | (branch,    |     | Control Signal|
                 |  return)    |     | Decoding      |
                 +-------------+     | (generates    |
                                     | control bits) |
                                     +---------------+
```

**Advantages:**
- Flexible: new instructions can be added by changing the microprogram (no hardware changes).
- Easier to design and debug (microcode is "software" in ROM).
- Suitable for CISC processors with large, complex instruction sets.
- Simplifies the control unit hardware (regular structure).

**Disadvantages:**
- Slower than hardwired control (extra level of indirection through control store).
- Control store access time adds to the instruction execution time.
- More power consumption.
- Requires more chip area (control store memory).

### 7. Timing Signals

The control unit operates synchronously with the system clock. The instruction cycle is divided into **clock cycles** (also called T-states or machine cycles).

**Typical timing diagram for a simple instruction (ADD):**

```
Clock:    _-_-_-_-_-_-_-_-_-_-_-_-_-_-_-_-_-_-_-_-_
          |   |   |   |   |   |   |   |   |   |   |
          | T0| T1| T2| T3| T4| T5| T6| T7| T8| T9|
          |   |   |   |   |   |   |   |   |   |   |
PCtoMAR:  __|^^|___________________________________
          |   |   |   |   |   |   |   |   |   |   |
MemRead:  ______|^^|^^|____________________________
          |   |   |   |   |   |   |   |   |   |   |
DataBus:  ______|???|^^|_data______________________
          |   |   |   |   |   |   |   |   |   |   |
MARtoMBR: ______|___|^^|___________________________
          |   |   |   |   |   |   |   |   |   |   |
MBRtoIR:  __________|___|^^|_______________________
          |   |   |   |   |   |   |   |   |   |   |
PCinc:    __________|^^|^^|________________________
          |   |   |   |   |   |   |   |   |   |   |
ALUop:    __________________|^^|^^|^^|_____________
          |   |   |   |   |   |   |   |   |   |   |
RegWrite: ______________________|___|^^|___________
```

**Legend:** `^^` = signal active (logic 1), `__` = signal inactive (logic 0), `??` = undefined/data settling.

- T0-T1: Fetch cycle (read instruction from memory).
- T2: Decode cycle (decode instruction, increment PC).
- T3-T5: Execute cycle (ALU performs ADD, result written to register).
- T6: Check for interrupts.

### 8. Comparison Table: Hardwired vs Microprogrammed Control

| Feature | Hardwired Control | Microprogrammed Control |
|---------|------------------|------------------------|
| **Speed** | Very fast (gate delays only) | Slower (control store access time) |
| **Flexibility** | Inflexible; changing instruction set requires hardware redesign | Flexible; new instructions added by changing microcode |
| **Design Complexity** | Complex, error-prone for large instruction sets | Simpler, more systematic design |
| **Cost (development)** | High (longer design and verification time) | Lower (microcode can be written and tested separately) |
| **Cost (production)** | Lower chip area (no control store) | Higher chip area (needs control store ROM) |
| **Power Consumption** | Lower | Higher |
| **Instruction Set** | Best for RISC (simple, fixed instructions) | Best for CISC (complex, variable instructions) |
| **Debugging** | Difficult (hardware changes needed) | Easier (microcode can be patched) |
| **Common Examples** | MIPS, ARM (older), modern high-performance CPUs | x86 (Intel, AMD), legacy CISC processors |
| **Implementation** | Combinational logic gates, flip-flops, decoders | Control store (ROM), microPC, microinstruction register |

### 9. Summary

- The control unit coordinates all CPU operations by generating control signals.
- **Hardwired control** uses physical logic gates to generate control signals -- fast but inflexible.
- **Microprogrammed control** uses microinstructions stored in control store memory -- flexible but slower.
- The control unit operates as a finite state machine, cycling through fetch, decode, execute, and interrupt states.
- Timing signals derived from the system clock ensure all operations occur in the correct sequence.

## Practice Problems

1. **Problem**: Explain the difference between hardwired and microprogrammed control in terms of how a new instruction (e.g., MULTIPLY) would be added to the CPU.
   **Answer**: In hardwired control, adding MULTIPLY would require redesigning the control logic circuit, adding new gates, and possibly modifying the decoder and sequencer. In microprogrammed control, you would only need to write a new microcode routine for MULTIPLY in the control store ROM -- no hardware changes are needed.

2. **Problem**: In which cycle does the control unit check for interrupts? What happens if an interrupt is pending?
   **Answer**: The control unit checks for interrupts at the end of the execute cycle (or after each instruction cycle is complete). If an interrupt is pending and interrupts are enabled, the control unit saves the current PC (return address) on the stack, loads the address of the Interrupt Service Routine (ISR) into the PC, and then begins fetching instructions from the ISR.

3. **Problem**: Given the control signals PCtoMAR, MemRead, MBRtoIR, and PCinc, write the sequence in which they should be asserted during the fetch cycle.
   **Answer**: The correct sequence is: (1) PCtoMAR -- place PC value onto address bus; (2) MemRead -- initiate memory read; (3) wait for memory access; (4) MBRtoIR -- transfer the fetched instruction from MBR to IR; (5) PCinc -- increment the Program Counter to point to the next instruction.

4. **Problem**: A microprogrammed control unit has a control store that can hold 1024 microinstructions, each 64 bits wide. What is the size of the control store in bytes?
   **Answer**: 1024 x 64 bits = 1024 x 8 bytes = 8192 bytes = 8 KB.

5. **Problem**: Compare the role of the sequence counter in a hardwired control unit versus the microPC in a microprogrammed control unit.
   **Answer**: The sequence counter in hardwired control tracks the current T-state (timing step) within the instruction cycle and feeds into the combinational logic to select the appropriate control signals. The microPC in microprogrammed control tracks the current microinstruction address within the microprogram routine stored in control store. Both serve to sequence the steps, but the sequence counter is hardware-based while the microPC reads from a memory array.
